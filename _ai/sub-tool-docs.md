# 子工具文档规范

r_lit 下每个子工具目录必须包含以下四个文档文件：

| 文件 | 语言 | 受众 | 风格 |
|------|------|------|------|
| `README.md` | **English** | 人类 | 可读性优先，含示例、使用场景 |
| `README_CN.md` | **中文** | 人类 | 与英文版对应，自然中文表达（非机翻） |
| `llms.txt` | **English** | AI/LLM | 结构化、无冗余叙述、字段式描述 |
| `llms_cn.txt` | **中文** | AI/LLM | 与英文版对应 |

## 语言规范（严格执行）

- `README.md` 和 `llms.txt` **必须全文英文**，不得出现中文
- `README_CN.md` 和 `llms_cn.txt` **必须全文中文**，不得出现英文段落（专有名词、命令、代码除外）
- 文件名后缀 `_CN` 代表中文版，无后缀代表英文版

## README（人类友好）

- 用自然段落描述功能、动机、使用场景
- 提供完整的安装和使用示例（可复制运行）
- 包含配置说明、注意事项、FAQ 等
- 适当使用 badge、表格、代码块增强可读性

## llms（AI 友好）

- 开头用一行概括工具用途
- 用扁平的 key-value 或短列表描述：名称、版本、输入、输出、依赖、CLI 参数
- 不写修饰性文字，只保留事实性信息
- 示例用最小可运行命令，不加解释段落

```
# 示例 llms.txt 结构
# <tool_name>
- purpose: ...
- version: ...
- install: ...
- cli_args: ...
- input: ...
- output: ...
- dependencies: ...
- usage_example: ...
```
