# Handoff · maquette v0.3 → v0.4

## Version Delivered
- Version: 0.3.0 (Toon look + undo/redo + shortcuts + palette schema v2)
- Date: 2026-04-23
- Session: Single chat session immediately following v0.2. Shipped all
  seven planned sub-tasks (A–G). No open decisions carried forward into
  v0.4 that weren't raised explicitly below.

## What Was Built

### User-visible
- **Toon / cel-shaded preview.** Every `CellCube` now renders with a
  custom 3-band cel shader (`assets/shaders/toon.wgsl`) plus a black
  inverted-hull outline via `bevy_mod_outline`. Cubes read as
  stylized / painterly rather than default-Bevy PBR.
- **View → Reset Preview** snaps the preview back to the default angle
  and zoom. Smooth transition, not a jump.
- **Undo / Redo** in the Edit menu; entries are per-cell, 256-deep ring
  buffer. Any brush stroke that crosses N cells yields N undo entries
  (stroke grouping deferred to v0.4).
- **Keyboard shortcuts** (Cmd on macOS / Ctrl elsewhere, transparently):
  - `⌘/Ctrl + N` — New Project… modal
  - `⌘/Ctrl + O` — Open…
  - `⌘/Ctrl + S` — Save
  - `⌘/Ctrl + Shift + S` — Save As…
  - `⌘/Ctrl + Z` — Undo
  - `⌘/Ctrl + Y` or `⌘/Ctrl + Shift + Z` — Redo
  - `⌘/Ctrl + R` — Reset Preview
  - `1`–`9` — select the nth palette swatch
- **Palette persisted inline** in `.maq` files (schema v2). v1 files
  open cleanly and default-palette-fill on load.
- **Help → About** now includes the "Preview is not the export" note so
  users understand the toon look won't 1:1 match their target engine.
- **Clear Canvas** now respects current canvas dimensions (was resetting
  to 16×16 regardless).

### Under the hood
- New module `src/toon.rs`: `ToonMaterial`, `ToonParams` (ShaderType
  packed `base_color` + `light_dir_bands` + `ambient_pad`), `ToonPlugin`.
- New WGSL shader `assets/shaders/toon.wgsl`: imports
  `bevy_pbr::forward_io::VertexOutput`, writes quantized diffuse at
  `@group(2) @binding(0)`.
- New module `src/history.rs`: `EditHistory` resource, `PaintOp`,
  `HistoryAction` message, `HistoryPlugin`. 256-deep ring buffer.
- `src/camera.rs`: explicit default yaw/pitch/radius constants, new
  `ResetPreviewView` message wired from UI.
- `src/project.rs`: schema bumped to 2, new `RgbaPayload` sRGBA-float
  on-disk format, backward-compatible v1 reader. `handle_project_action`
  clears history on New / Open. `Grid::paint/erase` now return
  `Option<(Cell, Cell)>` for history recording.
- `src/ui.rs`: new `handle_shortcuts` using
  `egui::Context::input_mut` + `KeyboardShortcut`. Undo/Redo menu
  entries guarded by `EditHistory::can_undo / can_redo`.
- Dependencies added: `bevy_mod_outline = "0.12"`.

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Toon technique in preview | Custom WGSL Material + `bevy_mod_outline` (inverted hull, ExtrudeFlat mode) | Balance of control (fragment shader is ours) and speed (outline is a mature crate). |
| Cel band count default | 3 bands, 35% ambient floor | Reads as toon without looking flat; tunable later via `ToonParams` if needed. |
| Light direction source | Hard-coded in shader uniform (`DEFAULT_LIGHT_DIR`), not driven by `DirectionalLight` entity | Product north star: hide light from the user. The scene's `DirectionalLight` is now cosmetically irrelevant to cell cubes. |
| Outline thickness | 3.0 logical pixels, black | Matches Kenney-style stylized look; revisit when user art-direction feedback arrives. |
| Undo granularity | Per-cell, no stroke grouping | Simpler data model; stroke grouping lands v0.4 once we have begin/end stroke UI signals. |
| Keyboard shortcut layer | `egui::Context::input_mut` + `KeyboardShortcut`, **not** Bevy's `ButtonInput<KeyCode>` | egui already owns input routing; using its API means text-field focus correctly suppresses digit shortcuts without extra guards. |
| Palette schema | v2 inline `Vec<RgbaPayload>` with `#[serde(default)]`; v1 loader auto-fills default palette | Per user lock. Forward-compat: reading v2 in a v1 build errors with `UnsupportedVersion`, which is correct. |
| Schema upper-version policy | Reject `version > SCHEMA_VERSION`, accept `1..=SCHEMA_VERSION` | Strict forward-compat boundary; users get a clear error instead of a silent downgrade. |
| `Edit → Clear Canvas` semantics | Preserve current width/height, wipe cells, clear history | Matches most raster editors; avoids a confusing resize. |

## Known Issues / TODO

- **Not user-verified visually**: cel bands, outline visibility, and
  stroke undo round-trip were smoke-tested (no panic + window creates)
  but not exercised by actual painting. Ask user to paint a few cells
  and confirm the toon look reads correctly at the start of v0.4.
- **Light direction** is fixed in shader; art-direction tuning will
  want a runtime uniform. Low priority — defer until a user asks.
- **`Edit → Clear Canvas` is not undoable**: once cleared, the old
  cells are lost. Fix is trivial (record a bulk op) but deferred to
  v0.4 alongside stroke grouping.
- **Digit shortcuts 1–9 only cover 9 swatches**; palettes with >9
  colors need `0` or mouse selection. The default palette has 12, so
  indices 10–12 are keyboard-inaccessible for now.
- **Initial window title flash** (`"Maquette — Asset Forge"`) was not
  addressed; reassessed as cosmetic, still tracked.
- **v0.2 A's file dialog concern** (sync modal blocks render) is still
  present; reassess if Windows/Linux testing complains.
- **`OutlinePlugin` ordering with Bevy's PBR passes on Metal** — smoke
  tests look clean on Apple M3 / macOS 26 but depth artifacts with
  transparent outlines are a known class of bug in outline crates.
  Keep an eye on it once painting happens.

## Next Version Contract

### Goal (one sentence)

Promote the tool from "N cubes per frame" to "one proper mesh per model"
(greedy meshing + vertex color), add palette customization UI, and
deliver the first real glTF export that opens cleanly in Godot, Unity,
and Blender.

### Deliverables
1. **Greedy meshing** of painted cells into a single `Mesh` with vertex
   colors per palette index. `bevy_mod_outline` still draws the preview
   outline from this single mesh.
2. **Canvas height** (Z-stack): click+drag or keyboard to raise/lower a
   cell's `height` field. Painter canvas shows a numeric overlay or
   shading gradient for stacked cells.
3. **Palette editor UI**: right-click a swatch to edit / delete; a "+"
   slot to add a new color. Persists via schema v2 (already in place).
4. **Stroke grouping** for undo: one drag = one undo entry. Requires a
   `BeginStroke` / `EndStroke` signal from the canvas interaction.
5. **First export**: `File → Export… → glTF` writes a `.glb` with
   a) the main mesh (geometry + vertex color + standard material),
   b) a baked **inverted-hull outline mesh** as a sibling, flat black unlit,
   c) a sidecar `README.txt` pointing at the per-engine guide docs.
6. **Per-engine guide docs** (`docs/export/godot.md`, `unity.md`,
   `blender.md`): half-page each, one working toon-material path.
7. **MAX_GRID bump** to 128 now that we're not spawning per-cell entities.

### Sub-tasks in order
- [ ] **A** Greedy mesher (`src/mesher.rs`) that replaces per-cell
      `CellCube` entities with a single `CellMesh` entity per rebuild.
- [ ] **B** Height model on `Cell`: UI slider or `[` `]` hotkeys,
      mesher honors it for vertical extrusion.
- [ ] **C** Palette editor UI + default-color-name-in-tooltip polish.
- [ ] **D** Stroke-grouping in `EditHistory`: `begin_stroke() / end_stroke()`,
      coalesce into one logical undo entry.
- [ ] **E** glTF export writer + inverted-hull baker (`src/export/gltf.rs`).
- [ ] **F** Per-engine docs (`docs/export/{godot,unity,blender}.md`).
- [ ] **G** MAX_GRID → 128 and regression-test on a full 128×128 canvas.

### Technology additions
- **`bevy_gltf` already transitive via DefaultPlugins** — but we need the
  **exporter** direction, not the importer. Options:
  - `gltf-json = "1"` + `gltf = "1"` (write path) — mature, 400-line max.
  - Hand-roll a minimal glTF 2.0 writer — only if the crate path proves
    too heavy for our single-mesh output. Default: use the crate.
- No other new dependencies expected.

### Open decisions that MUST be asked

1. **Export format scope for v0.4**:
   - Option 1 — glTF (`.gltf` + `.bin`) only. Most transparent, git-friendly.
   - Option 2 — GLB (single-file binary) only. Faster to ship, easier
     for users to drag into an engine.
   - **Option 3 (agent default)** — Both, user picks in the Export dialog.
     Let the crate do the work either way.

2. **Inverted-hull outline thickness / color in exports**:
   - User-configurable per-export (slider + color picker in the Export
     dialog)? Or hard-coded defaults (3%, black)?
   - **Agent default if user says "you pick"**: configurable, because the
     preview width and the export width will want to differ (preview is
     in pixels, export needs model-space percentage).

3. **Canvas height cap**:
   - How tall can a column be before it stops reading as "voxel asset"
     and starts reading as "minecraft tower"? Proposed hard cap 8. User
     can confirm or overrule.
   - **Agent default**: cap at 8.

### Exit criteria
- Painting 100% of a 128×128 canvas stays >45 fps on an Apple M3.
- `File → Export… → GLB` produces a file that:
  - opens in Blender 4.x without errors (drag-drop)
  - imports into Godot 4.x as a scene with two MeshInstance3D nodes
  - imports into Unity 6.x / URP as a prefab with two submeshes
  - preserves vertex colors (spot-check in each engine)
- `Ctrl+Z` after a full multi-cell drag reverts the entire drag as one
  undo entry, not N.
- Palette add/remove/recolor round-trips through save/load.
