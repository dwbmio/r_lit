# Group Vibe Workbench - 使用指南

## 快速开始

### 1. 启动应用

有两种方式启动应用：

#### 方式 A: 命令行指定昵称（推荐）

```bash
group_vibe_workbench launch --nickname "Alice"
# 或简写
group_vibe_workbench launch -n "Alice"
```

这样会直接使用你指定的昵称，跳过登录界面。

#### 方式 B: 启动后在 GUI 中设置

```bash
group_vibe_workbench launch
```

首次启动时，你会看到登录界面，可以：
- 点击"使用默认昵称"（自动生成 `User_xxxxxxxx`）
- 点击"开始使用"（使用当前输入的昵称，如果为空则使用默认）

**注意**: 由于 GPUI 0.2.2 的限制，GUI 中的文本输入功能有限，推荐使用命令行参数指定昵称。

### 2. 发现或创建群组

登录后，你会进入群组发现页面：

```
🔍 发现群组

📡
搜索本地网络中的群组

[🔍 搜索群组]  [➕ 创建新群组]
```

你有两个选择：

#### 选项 A: 搜索现有群组

1. 点击 **"🔍 搜索群组"** 按钮
2. 应用会扫描本地网络（5秒）
3. 发现的群组会显示在列表中：

```
发现 2 个群组:

┌─────────────────────────────────┐
│ 📁 group_1709123456             │
│ 3 个成员在线              [加入] │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ 📁 group_1709123789             │
│ 1 个成员在线              [加入] │
└─────────────────────────────────┘
```

4. 点击任意群组的 **"加入"** 按钮加入该群组

#### 选项 B: 创建新群组

1. 点击 **"➕ 创建新群组"** 按钮
2. 应用会自动生成一个群组 ID（例如：`group_1709123456`）
3. 你会立即进入群组大厅

### 3. 群组大厅

加入群组后，你会看到群组大厅界面：

```
🏠 群组: group_1709123456        你好, Alice

👥 成员列表 (3)

┌─────────────────────────────────┐
│ 👤 Alice                   👑   │
│ 在线                            │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ 👤 Bob                          │
│ 在线                            │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ 👤 Charlie                      │
│ 离线                            │
└─────────────────────────────────┘

📝 Shared Prompt Context
```

在这里你可以：
- 查看所有群组成员
- 看到谁在线/离线
- 看到谁是群组领导者（👑）
- 编辑共享的 Prompt 上下文文件（即将推出）

## 工作原理

### 零配置网络

Group Vibe Workbench 使用 **mDNS (Multicast DNS)** 进行本地网络发现：

- **无需配置**: 不需要输入 IP 地址或端口
- **自动发现**: 应用自动在本地网络中广播和发现其他节点
- **即时连接**: 发现后立即建立 P2P 连接

### P2P 协作

所有数据通过 **点对点 (P2P)** 方式同步：

- **无中心服务器**: 所有节点平等，无需中央服务器
- **CRDT 同步**: 使用 Automerge CRDT 确保数据一致性
- **自动领导者选举**: 使用 Bully 算法自动选举领导者
- **NAT 穿透**: 使用 iroh 实现 NAT 穿透和中继

### 数据存储

每个节点维护自己的数据副本：

- **本地文件**: `./prompt_context.txt`
- **Murmur 存储**: `./workbench_data/swarm/<user_id>/`
- **自动同步**: 本地更改自动同步到所有在线节点

## 使用场景

### 场景 1: 团队协作

**Alice 创建群组**:
```bash
# Alice 启动应用并指定昵称
group_vibe_workbench launch -n "Alice"

# 点击 "创建新群组"
# 群组 ID: group_1709123456
```

**Bob 加入群组**:
```bash
# Bob 在同一网络启动应用
group_vibe_workbench launch -n "Bob"

# 点击 "搜索群组"
# 看到 "group_1709123456 (1 个成员在线)"
# 点击 "加入"
```

**Charlie 加入群组**:
```bash
# Charlie 也在同一网络
group_vibe_workbench launch -n "Charlie"

# 点击 "搜索群组"
# 看到 "group_1709123456 (2 个成员在线)"
# 点击 "加入"
```

现在 Alice、Bob 和 Charlie 都在同一个群组中，可以协作编辑共享文件！

### 场景 2: 多群组环境

在同一网络中可以有多个独立的群组：

```
网络中的群组:
- group_1709123456 (Alice, Bob, Charlie)
- group_1709123789 (David, Eve)
- group_1709124000 (Frank)
```

每个群组的数据完全隔离，互不干扰。

## 故障排除

### 找不到群组

如果搜索不到群组，检查：

1. **网络连接**: 确保所有设备在同一局域网
2. **防火墙**: 确保防火墙允许 mDNS (UDP 5353)
3. **时间窗口**: 搜索只持续 5 秒，可能需要重试

### 无法加入群组

如果无法加入群组：

1. **检查日志**: 运行 `RUST_LOG=debug group_vibe_workbench launch`
2. **网络权限**: 确保应用有网络访问权限
3. **端口冲突**: 确保没有其他应用占用相关端口

### 成员显示离线

如果成员显示离线但实际在线：

1. **心跳超时**: 等待几秒让心跳机制更新
2. **网络延迟**: 检查网络连接质量
3. **重新连接**: 尝试退出并重新加入群组

## 高级用法

### 自定义窗口大小和昵称

```bash
group_vibe_workbench launch --width 1920 --height 1080 --nickname "Alice"
```

### 查看详细日志

```bash
RUST_LOG=debug group_vibe_workbench launch
```

### 清理数据

删除所有本地数据：

```bash
rm -rf ./workbench_data
rm -f ./prompt_context.txt
rm -f ./user.db
```
