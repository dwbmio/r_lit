# v0.10 D-1 — Material drawer + per-slot texgen — Complete

Status: **shipped on `main`** (this commit).

This handoff covers two slices that landed together:

* **D-1.A** — wire the schema-v4 `ProjectMeta` resource through the
  GUI (Open / Save / autosave / recovery), expose it in a new
  Material drawer in the left panel, and unify paint strokes + meta
  edits into a single LIFO `EditHistory`.

  right-click a swatch → `Generate texture ▶` → pick provider lane

  `~/.cache/maquette/textures/<sha>.png`; the slot's
  `PaletteSlotMeta::texture` is populated.

  Rustyme Canvas-group with chord callback).

  shader (so generated PNGs actually appear in the 3-D preview).

The Material drawer's `View: Flat / Textured` toggle is therefore
**persisted only** in this milestone — flipping it doesn't yet
alter what the preview camera renders.

#### Undo/redo across paint + meta

The old per-stroke undo stack was `VecDeque<Stroke>`. v0.10 D-1
generalised it to `VecDeque<EditEntry>` with two variants:

```rust
pub enum EditEntry {
    Paint(Stroke),
    Meta(MetaEdit),
}

pub enum MetaEdit {
    SetModelDescription { before: String, after: String },
    SetIgnoreColorHint   { before: bool,    after: bool },
    SetViewMode          { before: PaletteViewMode, after: PaletteViewMode },
}
```

`HistoryAction::Undo` / `Redo` walks back the unified timeline; a
paint stroke replays via `apply_cell`, a meta edit replays via
`apply_meta_undo` / `apply_meta_redo`. The user's mental model
("undo whatever I just did") is preserved regardless of whether
the latest action was a brush stroke or typing in the model
description.

`record_meta` skips identity flips (e.g. focus-loss on an
unchanged textarea) so an undo stack never sees `SetModelDescription
{ before: "x", after: "x" }`.


#### Cache layout

PNGs land at `~/.cache/maquette/textures/<sha256-of-request>.png`.
The cache key is computed from the full `TextureRequest`
(prompt + seed + width + height + model id), so flipping
the same slot between Mock and Fal stores both PNGs side by side
without collision.

`PaletteSlotMeta::texture` stores `TextureHandle { cache_key,
generated_at }` — no PNG bytes inline. The wgpu upload path that
will read these (D-1.D) does `texgen::cache_get(&dir, &request)`
or, more typically, reads `<cache_key>.png` directly off disk
since the generation parameters aren't kept on the slot.

#### Per-slot stable seed

```rust
fn stable_slot_seed(prompt: &str, slot: u8) -> u64 {
    let mut h = DefaultHasher::new();
    prompt.hash(&mut h);
    slot.hash(&mut h);
    h.finish()
}
```

Two slots whose derived prompts coincide still get different
seeds — the `slot_idx` is hashed in. Re-rolling the same slot
with an unchanged prompt is reproducible across launches; that's
the property the GUI's "Regenerate" button (future polish) needs.

## Manual verification

The user-facing `#D1` checks land in `docs/handoff/USER-TODO.md`:

1. Open the editor, expand the Material drawer, type a sentence,
   hit Cmd+S → reopen the file → description survives.
2. Type three new words → Cmd+Z → words disappear.
3. Paint a stroke, type a meta change, paint another stroke.
   Cmd+Z three times reverts in strict reverse order.
4. Right-click any palette slot → `Generate texture → Mock` →
   toast "Mock (offline) done · slot #N", file appears in
   `~/.cache/maquette/textures/`.
5. With `MAQUETTE_RUSTYME_REDIS_URL=...` set →
   `Generate texture → Rustyme CPU` → real PNG lands.
6. With Rustyme + Fal env (`MAQUETTE_RUSTYME_MODEL=fal-ai/flux/schnell`)
   → `Generate texture → Rustyme Fal` → AI-generated PNG.
