# Murmur 服务发现总结

## 问题：Murmur 如何找到 Group 列表？

### 答案：目前无法自动发现，需要手动连接

**当前状态**：
- ❌ **没有自动服务发现**
- ❌ **没有群组列表功能**
- ❌ **没有成员发现功能**
- ✅ **可以手动连接**（需要交换 NodeAddr）

### 当前连接方式

```bash
# 终端 1: 启动节点 A
cargo run --example group_chat alice
# 输出: 📍 Node Address: NodeAddr { node_id: abc123..., ... }

# 终端 2: 启动节点 B 并连接到 A
cargo run --example group_chat bob "NodeAddr { node_id: abc123..., ... }"
```

**问题**：
1. 必须手动复制粘贴地址
2. 不知道有哪些群组存在
3. 不知道群组里有谁
4. 无法列出在线节点

## 解决方案

### 方案对比

| 方案 | 适用场景 | 实现难度 | 工作量 | 优先级 |
|------|---------|---------|--------|--------|
| **本地网络发现 (mDNS)** | 局域网协作 | ⭐ 简单 | 1-2 天 | 🔥🔥🔥 推荐 |
| **中心注册服务器** | 互联网协作 | ⭐⭐ 中等 | 3-5 天 | 🔥🔥 实用 |
| **DHT 去中心化** | 大规模公开 | ⭐⭐⭐ 复杂 | 1-2 周 | ⭐ 完美但复杂 |

### 推荐方案：本地网络发现（mDNS）

**为什么推荐**：
- ✅ 实现简单（1-2 天）
- ✅ 零配置，自动发现
- ✅ 无需中心服务器
- ✅ 适合大多数使用场景（同一办公室/家庭网络）

**工作原理**：
```
节点 A 启动 → 广播到局域网 → "我是 Alice，在群组 cc"
节点 B 启动 → 搜索局域网 → 发现 Alice → 自动连接
```

**实现步骤**：
1. 添加 `mdns-sd` 依赖
2. 创建 `discovery.rs` 模块
3. 实现广播和发现功能
4. 在 UI 中添加"搜索群组"按钮

**使用效果**：
```rust
// 用户 A
swarm.advertise_local("Alice")?;  // 广播自己

// 用户 B
let groups = Swarm::discover_local_groups(5)?;  // 发现群组
// 输出: ["cc", "dev-team", "friends"]

let members = Swarm::discover_group_members("cc", 3)?;  // 发现成员
// 输出: [Alice, Bob, Charlie]

// 自动连接
for member in members {
    swarm.connect_peer(&member.node_addr).await?;
}
```
