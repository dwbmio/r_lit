# Handoff · maquette v0.2 → v0.3

## Version Delivered
- Version: 0.2.0 (Grid MVP + persistence + sized canvas + UX audit pass)
- Date: 2026-04-23
- Session: One chat session, continuing from the initial pivot decision and
  v0.2 Day 1 work. Shipped A, C, D of the v0.2 plan; B (toon shader) held
  at decision point per WORKFLOW.md §Decision-Point Discipline.

## What Was Built

### User-visible
- **2D paint canvas on the left, real-time 3D preview on the right.** Left-click
  to paint, right-click to erase, drag preview to turn, scroll to zoom.
- **12-color fixed palette** with selected-swatch highlight.
- **File → New…** opens a modal to choose canvas width × height (4–64).
- **File → Open… / Save / Save As…** read/write `.maq` JSON project files
  with native file dialogs (`rfd`).
- **Window title** shows project name + `•` prefix when unsaved.
- **Status bar** shows project name, painted count, and control hints
  (all phrased in user-domain terms — no "camera / light / mesh / UV").
- **Help → About Maquette** summarizes core interactions and controls.
- **Edit → Clear Canvas** resets the current project's cells.

### Under the hood
- New module `src/project.rs`: `ProjectFile` schema (v1), `CurrentProject`
  resource, `ProjectAction` message, `ProjectPlugin`.
- New `Grid::with_size(w, h)` clamps to `MIN_GRID` (4) ... `MAX_GRID` (64).
- `src/ui.rs` now owns a `UiState` resource for modal state (`show_about`,
  `new_project` draft); UI is thin and routes intents via messages.
- Dependencies added: `rfd = "0.17"`.
- `docs/handoff/` directory with `WORKFLOW.md`, `COST_AWARENESS.md`,
  `NEXT.md`, and this file.

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Project file format | JSON, schema `version: 1`, single pretty-printed file | Git-friendly, human-readable, trivial to extend. Compact binary is premature optimization at 64×64 max. |
| Palette persistence in v0.2 | Do NOT save palette colors, only `selected_color` index | v0.2 palette is fixed. v0.3 adds user palettes and bumps schema to v2 with migration. |
| File dialog library | `rfd` (Rusty File Dialog), sync variant | Native dialogs on all platforms; sync blocking main thread during modal is the expected OS behavior. |
| New Project flow | Modal with W/H sliders, not a "create then resize" flow | Resizing a painted canvas is a separate feature (v0.3 C). Creation-time decision keeps v0.2 simple. |
| Canvas size upper bound for v0.2 | 64×64 | Per-cell cube spawning stays under ~4k entities even at max; greedy meshing (v0.3) will lift this cap to 256. |
| UX vocabulary | "preview", "turn", "canvas", "project", "color" — NEVER "camera", "light", "mesh", "material", "UV" in user-facing strings | Product north star (see COST_AWARENESS.md §North Star). |
| Dirty tracking strategy | UI calls `current.mark_dirty()` on actual cell changes (using `Grid::set_if_changed` feedback via `grid.dirty` flip) | Avoids change-detection false positives from the rebuild system toggling `grid.dirty`. |
| Toon shader (v0.2 B) | **Deferred** — held at decision point | Needs user input on implementation path (see NEXT.md). |

## Known Issues / TODO

- `View → Reset Preview` is a placeholder; it shows a tooltip "Coming soon"
  but does not wire to the camera yet. First task of v0.3 D.
- No **undo / redo** yet. Cost-aware note: per COST_AWARENESS.md this is
  expected to be expensive; scheduled for v0.3 alongside the keyboard
  shortcut pass.
- No **autosave**. Safe to defer until after v0.3 toon work because users
  can now explicitly save.
- `File → Export…` still a placeholder (arrives in v0.6).
- Sync `rfd` modal freezes Bevy render for the dialog duration. Users
  don't notice on macOS native panels; reassess if Windows/Linux testing
  surfaces complaints.
- Initial window title `"Maquette — Asset Forge"` flashes for ~1 frame
  before `update_window_title` overwrites it to `"Untitled — Maquette"`.
  Cosmetic only.

## Next Version Contract

### Goal (one sentence)

Ship a distinctly **toon-shaded** preview — flat cel bands + outline —
and wire `View → Reset Preview` so the 3D look and camera feel intentional
rather than default-Bevy.

### Deliverables
1. Custom `Material` that renders cells with 3-band cel shading from a
   single directional light (hidden from the user; shader constant).
2. Outline pass (approach TBD — see decision below).
3. `View → Reset Preview` wired to a `CameraAction::Reset` message that
   returns `PanOrbitCamera` to its default azimuth/polar/radius.
4. Undo / redo for paint and erase operations (at least 32-depth ring
   buffer; no grouping yet, 1 stroke = N undo entries is acceptable for v0.3).
5. Keyboard shortcuts: `Ctrl+Z/Y`, `Ctrl+N/O/S`, `Shift+Ctrl+S`, digit keys
   `1`–`9` for palette selection.

### Sub-tasks in order
- [ ] **A** Toon material v0 (flat shading with 3 cel bands, no outline). WGSL custom material.
- [ ] **B** Outline pass — pick implementation in the decision below.
- [ ] **C** `CameraAction::Reset` + wire to `View → Reset Preview`.
- [ ] **D** Undo/redo model (`EditHistory` resource; record reverse-ops at paint/erase/clear).
- [ ] **E** Keyboard shortcuts (implement in a dedicated `input.rs` plugin).

### Technology additions
- **Custom WGSL material**: write `assets/shaders/toon.wgsl`, register via
  `Material` trait + `MaterialPlugin<ToonMaterial>`.
- **Optional**: `bevy_mod_outline` crate (pending decision below).
- No new runtime dependencies expected beyond the above.

### Open decisions that MUST be asked

1. **Toon outline implementation** (blocking v0.3 B):
   - Option 1 — custom WGSL post-process normal/depth edge detector
     (full control, ~1.5 days shader work, highest ceiling)
   - Option 2 — `bevy_mod_outline` crate (fastest to visible result,
     mesh-based inverted-hull style, lower ceiling)
   - Option 3 — hybrid: `bevy_mod_outline` first, replace with custom
     post-process if v0.4 benchmarks or aesthetic need it
   - **Agent default if user says "you pick"**: Option 3.

2. **Palette persistence in schema v2** (blocking any work that touches
   palette customization):
   - Option 1 — store full `Vec<Color>` inline in `.maq` (simple,
     redundant across files with same palette)
   - Option 2 — named external palette files `palette/*.maqpal`, project
     references one by name (requires palette library, bigger v0.3 scope)
   - Default: Option 1 for v0.3, promote to Option 2 in v0.4 if users ask.

### Exit criteria
- Preview visibly reads as toon / cel-shaded (side-by-side screenshot
  comparison against v0.2 looks clearly different).
- `Ctrl+Z` after painting reverts the last cell change; repeated `Ctrl+Z`
  walks back to a clean canvas.
- `Ctrl+S` saves without opening the File menu.
- All new systems obey the `ProjectAction` / `CameraAction` /
  `EditHistoryAction` message pattern — no direct resource coupling
  between UI and Bevy systems.
