## §1 — v0.9 polish (GUI ergonomics + macOS 26 export crash fix)

### Delivered

1. **Async export pipeline** (`src/export_dialog.rs` new,
   `src/export.rs` rework). The previous code called
   `rfd::FileDialog::save_file()` synchronously inside an
   `EguiPrimaryContextPass` system. On macOS 26+, that path ends in
   `NSSavePanel.runModal()`, which nests a modal run-loop under
   winit's own callback — the two compete for the main-thread
   dispatch queue and the app wedges. Symptom in logs:
   `ui: opening save dialog for export` followed by silence.
   Fix: switch to `rfd::AsyncFileDialog` driven through
   `AsyncComputeTaskPool::spawn`. The actual export job also moved
   off the main thread; the GUI shows a centered "Exporting…" modal
   with a per-frame `elapsed Xs` counter while the file is being
   written. `ExportInFlight` resource greys the menu entry while
   running and drops redundant `ExportRequest` events with an
   `export: ignoring request` log line.

2. **Brush floating HUD + Paint Mode (Overwrite / Additive)**
   (`src/ui.rs`). The brush controls left the bottom of the side
   panel and live as a floating Blender-style overlay in the canvas
   top-left. Adds an `Overwrite ↔ Additive` toggle (button + `A`
   key + Edit menu). In Additive mode, dragging a stroke adds the
   brush height to each cell at most once per stroke (re-uses the
   existing `stroke_touched` guard so a 5-cell drag doesn't pile up
   60+ height). Status-bar text mirrors the active mode.

3. **Right-click = cycle block shape** + **Backspace / Delete = erase**
   + **height number badge** + **2-D sphere ring marker**
   (`src/grid.rs`, `src/mesher.rs`, `src/preview_mesh.rs`,
   `src/ui.rs`). New `grid::ShapeKind { Cube, Sphere }` enum, sits
   on `Cell::shape` with `#[serde(default)]` for full backward
   compatibility with pre-v0.9 `.maq` files. Right-click on a
   painted cell cycles the shape; right-click on an empty cell is
   a no-op; a right-button drag cycles each cell at most once per
   stroke. Backspace / Delete erases the hovered cell (one undo
   step per press). 2-D canvas paints a small ring marker on
   sphere cells; height number ≥ 2 shows as a small badge on the
   cell's top-left (auto-hidden when cell width < 12 px). **Cube
   mesher is strictly cube-only** — `build_color_buckets` (greedy),
   the culled oracle, `max_column_height`, `voxel_color`, and
   `neighbor_filled` all route through `is_cube_voxel`. Sphere
   cells render through `build_sphere_instances(grid) →
   Vec<SphereInstance>` (one `Sphere(0.5)` entity per column,
   scaled by height). Exporter / CPU rasterizer skip sphere cells
   — shape-aware glTF / PNG is the post-v1.0 follow-up. Three new
   mesher tests cover sphere exclusion, non-culling of a cube
   adjacent to a sphere, and `cycle_shape` no-op on empty cells.

4. **PIP click → main camera animation** + **PIP border colour
   coding + axis compasses** (`src/multiview.rs`, `src/camera.rs`).
   Hovering a PIP turns its border light-blue and the cursor into a
   pointer; clicking smoothly animates the main camera's yaw / pitch
   to that view's angle (projection stays perspective). Borders
   colour-code the missing axis: Top = green, Front = blue, Side =
   red. Each PIP draws a small tinted-disc compass in its top-right
   corner. World axes rendered when `View → Show World Axes` is on
   are now ~55 % alpha so they don't paint over the model surface.

5. **Float window pose memory + close-button parity**
   (`src/float_window.rs`). Closing the float window's OS button
   now copies the float camera's pose back to the docked camera, so
   the next interaction continues from the same angle. Reopening
   the float restores the *floating* pose, not the original docked
   pose. The `Float` toolbar button always reflects actual state.

6. **Preview zoom buttons + keyboard shortcut** (`src/ui.rs`,
   `src/camera.rs`). The floating preview toolbar grew `−` / `+`
   buttons (left of `Fit` / `Reset`). `+` / `=` zooms in, `-` zooms
   out. Both clamp to `[MIN_RADIUS=3, MAX_RADIUS=120]`. Wheel
   zooming continues to work as before.

7. **Event-driven rendering** (`src/main.rs`). `WinitSettings::desktop_app()`
   replaces the implicit `WinitSettings::game()`. Idle CPU drops to
   ~0 % — the same model Godot, Blender and other DCC editors use.
   Smooth animations (Fit / Reset / PIP click / brush stroke) work
   because `camera::request_redraw_while_animating` pumps
   `RequestRedraw` every frame while `PanOrbitCamera` is still
   converging. Cmd-Tabbing away drops to `ReactiveLowPower(60s)`.

8. **Modal centering (New / Export / About / palette delete /
   recover)** (`src/ui.rs`, `src/session.rs`). All menu-launched
   modals previously defaulted to the egui top-left, ~(32, 32),
   where they overlapped the menu bar + left side panel + brush
   HUD — users reported "click does nothing" because the actual
   click target was hidden under the side panel. Fixed by pinning
   every modal `egui::Window` to `Align2::CENTER_CENTER`,
   non-draggable.

9. **Window resize floor** (`src/main.rs`).
   `WindowResizeConstraints { min_width: 1000.0, min_height: 640.0
   }`. Below this the layout overlaps (PIP strip eats the side
   panel). Floor was picked to fit the side panel min (360) + a
   ~300 px main preview slot + the PIP strip min (~324) without
   collision.

10. **Egui primary-context anchoring** (`src/main.rs`,
    `src/camera.rs`). `EguiGlobalSettings { auto_create_primary_context:
    false }` plus an explicit `PrimaryEguiContext` component on the
    main preview camera. Without this, a Startup-ordering race lets
    a PIP camera grab the primary context and the entire UI
    collapses into a 180×180 square in the bottom-right corner.

### Notable bug fixes (already merged into the items above)

* macOS 26 NSSavePanel deadlock under winit (item 1).
* PIP-camera-wins-Startup-race UI collapse (item 10).
* "Click does nothing on New / Export / About" hidden behind side
  panel (item 8).
* Float-window orbit pose lost on close (item 5).
* `bevy 0.18` material bind group migration to `@group(3)` —
  shader updated to `@group(#{MATERIAL_BIND_GROUP})` preprocessor
  macro (`assets/shaders/toon.wgsl`).

## §2 — v0.10 A — texgen module + MockProvider + disk cache + CLI

### Delivered

1. **`maquette::texgen` lib module** (`src/texgen.rs`). Trait-based
   texture-generation API, sync on purpose so the GUI offloads via
   `AsyncComputeTaskPool::spawn(async move { provider.generate(...) })`
   without dragging a tokio runtime through the lib. Determinism
   contract: same `TextureRequest` → byte-identical PNG output, so
   the disk cache key is meaningful and tests are reproducible.

2. **`MockProvider`** — deterministic, offline, zero external deps.
   Hashes (prompt, seed, model) → SplitMix64 noise field over a
   prompt-derived base colour. Caps at 1024×1024 to keep CI honest.
   Used by tests, by the CLI when you don't want to bill an API,
   and as a placeholder while choosing whether to spend real money.

3. **Disk cache** — `default_cache_dir()` returns
   `$XDG_CACHE_HOME/maquette/textures/` then falls back to
   `$HOME/.cache/maquette/textures/`. Filenames are `<key>.png`
   where `<key>` is the 64-char SHA-256 of the request, prefixed
   by the domain separator `b"maquette-texgen-v1\x00"`. Cache miss
   surfaces as `Ok(None)`, not an error, so callers cleanly fall
   through to the provider.

4. **`maquette-cli texture gen` subcommand**
   (`src/bin/maquette_cli.rs`). Args: `--prompt / --seed / --width
   / --height / --model / --provider {mock,rustyme} / --no-cache /
   -o`. Logs `wrote <path> (<bytes>, provider=<name>,
   cache_key=<hex>)` so the user can grep cache hits across runs.

### New public API

* `maquette::texgen::TextureRequest` (struct: prompt / seed / width
  / height / model)
* `maquette::texgen::TextureBytes` (newtype around `Vec<u8>` PNG
  bytes)
* `maquette::texgen::TextureProvider` (trait)
* `maquette::texgen::ProviderError` (`thiserror`)
* `maquette::texgen::MockProvider` (struct with `MODEL_ID =
  "mock-v1"`)
* `maquette::texgen::default_cache_dir() -> Option<PathBuf>`
* `maquette::texgen::cache_get` / `cache_put`

### Tests added

* `cli_texture_gen_mock_is_deterministic` — same args twice → byte-
  identical PNG.
* `cli_texture_gen_diverges_on_prompt` — different prompt → different
  bytes.
* `cli_texture_gen_cache_round_trip` — first run misses cache, second
  run with same args reads from disk.

## §3 — v0.10 B — RustymeProvider + revoke/purge CLI + frozen worker contract

### Delivered

1. **`maquette::texgen::rustyme::RustymeProvider`**. Sync Redis
   producer (`redis = "0.27"`, no tokio). On `generate()`:
   1. Build a `TaskEnvelope { id: uuid_v4, queue: "texgen", task:
      "texture.gen", kwargs: { prompt, seed, width, height, model,
      cache_key } }`.
   2. `LPUSH rustyme:texgen:queue <json>`.
   3. `BRPOP rustyme:texgen:result <timeout>`.
   4. Decode `{"png_b64": "..."}` from the wrapped `{task_id,
      status, result, ...}` payload.

2. **`maquette-cli` extensions** (`src/bin/maquette_cli.rs`). New
   verbs: `texture revoke --task-id <id>` (POST to
   sonargrid admin), `texture purge --queue <name>` (POST to
   sonargrid admin). Existing `texture gen --provider rustyme`
   wires up to `RustymeProvider`.

3. **Configuration via env vars** (per project convention):
   * `MAQUETTE_RUSTYME_REDIS_URL` (required for `--provider rustyme`)
   * `MAQUETTE_RUSTYME_ADMIN_URL` (required for `revoke` / `purge`)
   * `MAQUETTE_RUSTYME_QUEUE_KEY` (default `rustyme:texgen:queue`)
   * `MAQUETTE_RUSTYME_RESULT_KEY` (default `rustyme:texgen:result`)
   * `MAQUETTE_RUSTYME_RESULT_TIMEOUT_SECS` (default `60`)

4. **Frozen worker contract** — `docs/texture/rustyme.md` is now
   the authoritative wire protocol. Maquette is the only producer
   today; any envelope-shape change must bump there first.

5. **sonargrid worker implementation roadmap** —
   `docs/texture/rustyme-worker-roadmap.md`. Stage 1 = Echo worker
   (~half day, unblocks `#TEX-B`). Stage 2 = real Fal.ai FLUX
   schnell call (~1 day). Stage 3+ = production polish (cost
   tracking, in-memory LRU, etc.).

### New public API

* `maquette::texgen::rustyme::RustymeProvider` (sync, `new(config)`)
* `maquette::texgen::rustyme::RustymeConfig` — fields cover redis
  URL, queue key, result key, admin URL, task name, result timeout,
  max retries. Constructed via direct field literal or
  `RustymeConfig::from_env()` (which honours
  `MAQUETTE_RUSTYME_*` env vars).

The wire envelope (`TaskEnvelope` / `ResultEnvelope` / `TextureResult`)
is intentionally **private** to the module. Workers should copy the
JSON shape from `docs/texture/rustyme.md`, not depend on
`maquette` as a library — keeps the worker side decoupled and lets
Maquette evolve its internal types freely.

### Tests added

* `live_round_trip_against_running_rustyme` — `#[ignore]` by
  default; runs only when `MAQUETTE_RUSTYME_LIVE_TEST=1` and a real
  sonargrid + Stage 1 worker are running.
* `cli_texture_gen_rustyme_requires_redis_url` — graceful error
  when env var is missing.
* `cli_texture_revoke_requires_admin_url`,
  `cli_texture_purge_requires_admin_url` — same for the admin
  endpoints.

### Known gap

`#TEX-B` end-to-end verification is **blocked on sonargrid Stage 1
Echo worker**. The producer side is fully shipped and unit-tested.
See `docs/texture/rustyme-worker-roadmap.md` § "Stage 1" for the
half-day shape of the unblocking work.
