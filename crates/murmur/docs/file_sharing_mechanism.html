5. 核心要点

维度 机制
数据模型 一次 put_file 产生 5 个 KV put，每个都独立走 CRDT → 广播 → ACK 链路
同步方式 每个 put() 立即 broadcast(CrdtUpdate) 给所有已连接 peer，非攒批
冲突解决 底层 Automerge CRDT 自动 merge (last-writer-wins for scalar)；上层 put_file_with_version 提供乐观锁
可靠传输 seq + ACK + 重传（3s 超时，最多 5 次），无 ACK 则丢弃
事件通知 DataSynced 在收到 CrdtUpdate / SyncResponse 时发射；PeerConnected 在连接建立时发射
局限 文件 ≤ 10MB；list_files() 和 audit_trail() 未实现；文件内容内联在 CRDT 消息中传输
