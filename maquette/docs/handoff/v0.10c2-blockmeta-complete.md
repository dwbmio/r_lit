# v0.10 C-2 — Block Library: BlockMeta + LocalProvider + HfrogProvider + GUI panel

**Shipped 2026-04-28 (morning).** A second wave on top of v0.10 C-1's
lib-side schema work. Adds the *content* layer: every "block" the
user can paint with now has a stable id, a name, a description, a
shape hint, a default colour and — critically — a `texture_hint`
that feeds straight into the texgen prompt-derivation chain. Twelve
blocks ship in the binary as the bundled "LocalProvider"; an
HTTP-backed `HfrogProvider` pulls additional blocks from the
artifact server at `https://starlink.youxi123.com/hfrog` and caches
them on disk.

This makes "块的种类" a first-class concept (rather than the old
binary `Cube | Sphere` shape enum), and prepares for v0.10 D-1's
single-prompt fan-out: instead of a plain RGB colour driving the
per-slot prompt, a bound block contributes a richer
`texture_hint` that the worker can actually act on.

## §1 — Why this exists

Until now, a Maquette "block" was:

```rust
pub enum ShapeKind { Cube, Sphere }
struct Cell { color_idx: Option<u8>, height: u8, shape: ShapeKind }
```

That captures geometry and *nothing else* — no name, no description,
no provenance, no idea what kind of surface the block is supposed to
look like. So when v0.10 D-1 tries to "make a Minecraft-style grass
dirt block", every per-slot prompt has to be fabricated from
`(model_description, RGB colour)` alone. That's adequate for "the
green slot is grass-coloured", but loses the natural language
("草地块顶面：杂乱草丛", "rich brown dirt with scattered pebbles")
that turns a procedural texture into a recognisable surface.

`BlockMeta` is the missing description layer. A slot can now bind
to a block id — the GUI gets a richer hover card, the texgen
pipeline gets a real prompt fragment, and the same data round-trips
to the project file so the binding survives Open / Save.

## §2 — New module: `maquette::block_meta`

### Types

```rust
pub struct BlockMeta {
    pub id: String,                       // slug, [a-z0-9_]
    pub name: String,                     // user-facing label
    pub description: String,              // hover-card text
    pub shape_hint: ShapeKind,            // default shape for new cells
    pub default_color: RgbaColor,         // fallback colour
    pub texture_hint: String,             // prompt fragment for texgen
    pub tags: Vec<String>,                // category filters
    pub source: BlockMetaSource,
    pub preview_s3_key: Option<String>,   // hfrog-side pre-render PNG
}

pub enum BlockMetaSource {
    Local,
    Hfrog { pid, name, ver, md5, fetched_at },
}

pub struct RgbaColor { r, g, b, a }

pub trait BlockMetaProvider {
    fn name(&self) -> &'static str;
    fn list(&self) -> Result<Vec<BlockMeta>, BlockMetaError>;
    fn get(&self, id: &str) -> Result<BlockMeta, BlockMetaError>;
}
```

### Bundled providers

* **`LocalProvider`** — 12 blocks, hardcoded in the binary, ids
  matching the default palette one-to-one so a fresh project's
  slot-N can sensibly bind to block-N (slot 0 ↔ `brick`, slot 1 ↔
  `lava`, slot 2 ↔ `sand`, slot 3 ↔ `grass`, slot 4 ↔ `ice`, slot 5
  ↔ `water`, slot 6 ↔ `amethyst`, slot 7 ↔ `wood`, slot 8 ↔
  `dirt`, slot 9 ↔ `stone`, slot 10 ↔ `bone`, slot 11 ↔ `moss`).
  Bilingual names (`"草地块 / Grass"`) for the in-house Chinese
  workflow plus international audiences.

* **`HfrogProvider`** — sync HTTP client (ureq, no tokio, matches
  the lib's "stay headless / sync at the boundary" rule from
  texgen). Hits hfrog endpoints:
  - `GET /api/artifactory/list?runtime=maquette-block/v1` → catalog
  - `GET /api/artifactory/find?name=&runtime=` → single block
  - `GET /api/artifactory/get_object_presigned_url?id=&action=Download`
    → presigned URL for an optional pre-rendered PNG preview (the
    second leg of `fetch_preview_png`).

### Wire schema (hfrog-side)

The Maquette block payload lives in hfrog's existing `tag` JSON
column. Schema:

```jsonc
{
  "tag": {
    "id": "grass",                                  // optional, fallback = artifact.name
    "display_name": "草地块 / Grass",                // optional, fallback = id
    "description": "...",                           // optional, fallback = artifact.descript
    "shape_hint": "cube",                           // "cube" | "sphere", default cube
    "default_color": [0.45, 0.80, 0.40],
    "texture_hint": "patchy minecraft-style grass top, ...",
    "tags": ["natural", "ground"],
    "preview_s3_key": "blocks/grass-preview.png"    // optional
  }
}
```

Bumping any field (renaming, adding a *required* one) requires
bumping the runtime to `maquette-block/v2`, mirror-promoted in
`MAQUETTE_HFROG_RUNTIME` env var.

### Disk cache

`~/.cache/maquette/blocks/{local,hfrog}/<runtime>/<id>.json` plus
optional `<id>.png` for pre-rendered previews. Atomic writes via
`<id>.json.tmp` + `rename` (same shape as the texgen cache). Cache
list tolerates malformed entries (logs + skips) so a single bad
file doesn't poison `block list`.

## §3 — `PaletteSlotMeta::block_id`

Schema-level: a single optional `String` field on the existing
`PaletteSlotMeta` (which itself was added in v0.10 C-1).
`#[serde(default)]` so v0.10 C-1 `.maq` files load unchanged, and
the field is `skip_serializing_if = Option::is_none` so a project
that never binds a block has zero on-disk overhead.

API: `Palette::set_block_id(idx, Option<String>) -> Option<Option<String>>`,
returning the previous value for the (D-1) undo-stack hookup.
Whitespace-only ids collapse to `None`, matching the existing
`set_override_hint` policy.

## §4 — `derive_texture_prompt` — single source of truth

Every caller (CLI, GUI Generate button, batch tools) goes through
`maquette::texture_meta::derive_texture_prompt`:

```rust
pub fn derive_texture_prompt(
    model_description: &str,
    slot: &PaletteSlotMeta,
    slot_color: Color,
    block: Option<&BlockMeta>,
    prefs: &TexturePrefs,
) -> String;
```

Priority chain (highest first):

1. **`slot.override_hint`** — explicit user input wins.
2. **`block.texture_hint`** combined with `model_description` as
   `"<model>, <hint>"`.
3. Block's `description` (fallback when `texture_hint` is empty).
4. **Pre-block fallback**: `model_description + ", " +
   "base color rgb(r, g, b)"`. The `base color` token suppresses
   when `prefs.ignore_color_hint` is set.
5. **Defensive default**: `"colored block"` so the worker never
   sees an empty prompt (texgen-fal would reject one outright).

Seven unit tests pin every leg of the priority chain.

## §5 — CLI: `maquette-cli block`

Three new verbs:

```sh
maquette-cli block list [--source local|hfrog|all] [--json]
maquette-cli block get <id> [--source ...] [--json]
maquette-cli block sync [--base-url URL] [--runtime NAME] [--json]
```

### Source semantics

* `--source local` — bundled set only, never hits the network.
* `--source hfrog` — hits the network unless the cache has a
  matching record.
* `--source all` (default) — bundled + cached hfrog (no network).
  hfrog wins on id collisions.

The CLI prints a fixed-width five-column table by default:

```
ID             FROM   SHAPE  COLOR      TEXTURE HINT
amethyst       local  cube   #a666d9    amethyst crystal cluster, faceted purple gems, low-poly
…
12 blocks total
```

`--json` emits `BlockMeta` records as a JSON array (suitable for
`jq | xargs maquette-cli block get`).

### Tests

Six new integration tests in `tests/cli.rs` (alongside the existing
texture / palette / export ones):

* `cli_block_list_local_table_form` — shape of the human output.
* `cli_block_list_local_json_is_valid` — JSON path round-trips.
* `cli_block_get_known_id_text_form`
* `cli_block_get_unknown_id_fails` (non-zero exit + actionable stderr)
* `cli_block_sync_against_in_process_mock_writes_cache` —
  spins up a TcpListener mock, drives `block sync --base-url
  http://127.0.0.1:N`, asserts the cache file lands on disk and
  the JSON shape is right.
* `cli_block_list_all_returns_local_when_hfrog_offline` — pins the
  contract that `--source all` never errors on a network outage.

### Configuration

| env var                          | default                                  |
|----------------------------------|------------------------------------------|
| `MAQUETTE_HFROG_BASE_URL`        | `https://starlink.youxi123.com/hfrog`    |
| `MAQUETTE_HFROG_RUNTIME`         | `maquette-block/v1`                      |
| `MAQUETTE_HFROG_TIMEOUT_SECS`    | `15`                                     |
| `MAQUETTE_NO_CACHE`              | (unset) — set to `1` to force network on `list` |

> ⚠️ **Port collision note.** hfrog's default bare-host port is
> `12121`, the same as sonargrid's Admin UI. When co-locating both
> services on one machine, configure them onto distinct ports —
> `https://starlink.youxi123.com/hfrog` mounts hfrog under a path
> prefix on a regular HTTPS port, which is the safer pattern.

## §6 — GUI: right-side Block Library panel

A new right `SidePanel` (`#block_library_panel`, default 280 px,
min 220 px) added to `ui_system`. Layout:

```
┌─ Block Library ──────────────────────┐
│  [Sync hfrog]              12 blocks │
│  ────────────────────────────────────│
│  Selected slot: #3 (bound to grass)  │
│                                      │
│  ┌───────────────────────────────┐   │
│  │ ▮  草地块 / Grass             │   │
│  │    id: grass · local           │   │
│  │    natural · ground            │   │
│  │    patchy minecraft grass top… │   │
│  │    [✔ Bound]      [Unbind]     │   │
│  └───────────────────────────────┘   │
│  …                                   │
└──────────────────────────────────────┘
```

Each card:
* 28 px swatch using `default_color`.
* Bilingual `name` (or `id` fallback).
* Tags on a dim row (when present).
* `texture_hint` italicised.
* `[Bind to selected slot]` button — disabled when already bound.
* `[Unbind]` shown when the selected slot is currently bound to
  this block.

Top-of-panel `[Sync hfrog]` button:
* Spawns a background `AsyncComputeTaskPool` task running
  `HfrogProvider::sync()` (same pattern as the export job).
* Greys out + relabels to `[Syncing…]` while the task runs.
* On success, merges new records into `library.blocks` and emits
  a toast `Synced N hfrog block(s)`.
* On failure, leaves the previous list intact and surfaces the
  error string in a red sub-label *and* a toast.

### Slot binding paths

Two ways to bind:
1. **Right-click a palette swatch** → context menu now has a
   "Bind block…" submenu (or `"Block: <id>"` when bound) listing
   every known block, with `✔` next to the current binding and an
   `Unbind` entry at the bottom when one exists.
2. **Click `[Bind to selected slot]`** on any library card.

### Visual cue

Bound slots get a small 5 px blue circle in the bottom-right
corner of their swatch — at-a-glance signal that this slot has a
block-mediated identity, not just a colour.

### Wiring

`block_library.rs` (new file in the GUI binary) owns:
* `BlockLibraryState` resource — `Vec<BlockMeta>` mirror, sync
  in-flight flag, last-error string.
* Three messages: `BlockBindAction { Bind | Unbind }`,
  `SyncBlockLibrary` (trigger), `PendingSync` (background task).
* Three systems: `populate_initial_library` at Startup,
  `handle_bind_action` (applies through `Palette::set_block_id`,
  marks project dirty, emits a toast, ends any open stroke so a
  Ctrl+Z doesn't fold the bind into a paint stroke),
  `handle_sync_request` + `poll_pending_sync` for the async task.

The actual *undo* of a bind / unbind is intentionally deferred to
v0.10 D-1's EditHistory enum widening — `set_block_id` already
returns the previous value, so wiring it is a cheap follow-up
when D-1 lands.

## §9 — What's deliberately *not* in this commit

* **Preview thumbnails.** `preview_s3_key` is captured on the wire
  schema and `HfrogProvider::fetch_preview_png` is implemented +
  cached, but the GUI panel doesn't decode and display the bytes
  yet. egui's image-loading wants a `&'static [u8]` or a wgpu
  texture handle — wiring that for an arbitrary `PathBuf →
  egui::TextureHandle` flow is its own commit. Cards currently
  show the swatch only; thumbnails plug in cleanly when they're
  built.
* **Per-cell block meta.** A block id binds to a *palette slot*,
  not a *cell*. Per-cell binding (every cell stores its own block
  id) would be the Minecraft-style take, but it's at odds with
  Maquette's "palette-first" philosophy. If users ask for it
  later, the schema gains a `Cell::block_id_override:
  Option<String>` field with the same `serde(default)` pattern.
* **Generate button + prompt-derivation hookup.** `derive_texture_prompt`
  exists and is unit-tested, but the GUI Generate button still
  uses the older code path. Wiring it is part of v0.10 D-1, which
  also picks up:
  - autosave migration to `write_project_with_meta` (still owed
    from C-1, see `v0.10c1-complete.md` § 7 — drop a typed
    description-edit and an autosave between two keystrokes
    silently overwrites it on disk),
  - `EditHistory` enum widening so block bind / unbind / hint
    edits hit the undo stack as proper events.
* **`block publish` CLI verb.** Maquette is read-only against
  hfrog. Publishing new blocks is an ops task using hfrog's own
  HTTP API; if the user wants a maquette-side publisher we add
  that as part of `block sync --upload` later.
