# v0.4 — meshing + height + glTF export

**Status:** shipped (core). Two stretch items (C palette editor,
D stroke-group undo) demoted to **v0.6** per the 2026-04-23 scope
change; see `COST_AWARENESS.md` §The Headless Invariant and
`NEXT.md` §Updated roadmap for context.

## User-visible

- **Culled single-mesh preview.** The old per-cell `CellCube`
  implementation is gone; the preview now spawns one `CellMesh`
  entity per palette color actually painted. Cross-color seams
  remain visible (they help the cel look), interior faces do not.
  A 128×128 canvas at full height 8 is interactive.
- **Brush height.** Left panel has a "Height" slider, 1..=8 cells.
  Painting a cell extrudes it vertically by that many cells. The
  cap of 8 is a **product decision**, locked in
  `COST_AWARENESS.md` §Post-v1.0 Possible Product Line Split —
  going higher invites a fork into a MagicaVoxel-shaped product
  we've explicitly declined to build pre-v1.0.
- **Canvas up to 128×128.** `MAX_GRID` bumped from 64, made
  tractable by the new mesher.
- **`File → Export…`** (and `Cmd/Ctrl+E`). Modal picks
  `.glb` vs `.gltf`, outline on/off, outline width (0–10% of
  bounding diagonal), outline color. Writes a boring,
  engine-ready glTF 2.0 file with one primitive per palette
  color (unlit via `KHR_materials_unlit`) and an optional
  inverted-hull outline mesh.
- **Three engine integration docs** under `docs/export/`:
  Godot 4, Unity 6, Blender 4. Each gives install, scene
  structure, outline handling, and troubleshooting.
- Updated **About** dialog and **Help** menu reflect the new
  Export entry point; the "Preview ≠ Export" disclaimer stays.

## Under the hood

- New module `src/mesher.rs` (~300 lines + 4 tests). Culled
  per-face meshing walks the implicit voxel grid, emits one
  bucket per distinct palette color, translates to world
  coordinates once after the walk. True greedy-rectangle
  merging is **not yet** implemented — deferred to v0.6. The
  current algorithm already collapses the ECS entity count
  from 131k to ≤ palette size, which is what matters.
- New module `src/export.rs` (~700 lines + 5 tests). Hand-rolled
  glTF 2.0 JSON types (no `gltf-json` crate dependency; writes
  the exact subset we need). GLB packs JSON + BIN chunks per
  spec §4.4; .gltf writes text + sibling .bin. Both paths share
  the same builder.
- Inverted-hull outline is a **geometry** effect: vertex
  positions extruded along normals by `width_pct% * diagonal`,
  triangle winding reversed. The engine's default backface
  culling turns that into a silhouette. No shader, no
  post-process, no engine help needed.
- `Cell::height` is now wired end-to-end (previously a reserved
  field). Legacy v1 project files that wrote `height: 0` are
  still loaded correctly — the mesher treats `0` as `1` so old
  canvases don't render as empty.
- Dev-dep `gltf = "1"` added for a round-trip test: we parse our
  own GLB output with the canonical parser and verify node /
  primitive / material counts. Catches any future spec drift
  (alignment, accessor offsets, unknown extensions).

## Decisions locked this iteration

- Outline implementation on export is **inverted-hull only** (a
  geometry effect). Shader-based outlines are an engine concern
  and we ship per-engine docs telling the user how to swap in
  their preferred approach.
- Outline width is expressed as a **percentage of the model's
  bounding diagonal**. This scales sensibly regardless of canvas
  size. The range is clamped to `0..=10` in the UI.
- Palette colors round-trip through `.maq` schema v2; the
  exporter reads palette directly so any edits to the palette
  (once v0.6 C lands) will be reflected in future exports.
- **Canvas column height cap = 8** (see
  `COST_AWARENESS.md`). This was a *product* decision, not a
  technical one; the mesher could easily handle much more.

## Known issues / caveats

- **`rfd` save dialogs block the main thread.** Acceptable on
  macOS/native; not yet validated on Windows or Linux. If it
  becomes a problem we can switch to `rfd::AsyncFileDialog`, but
  the rest of the app's save/open/export path depends on it
  synchronously so that's a bigger change.
- **Outline seams at color boundaries.** Each per-color mesh
  owns its own `OutlineVolume` in the preview; this produces a
  thin black seam at color boundaries. The export baker does
  not have this issue (it merges all buckets into one outline
  hull). Preview seams are acceptable and actually read as a
  cel effect; a single-volume preview outline is a v1.0-ish
  polish item.
- **Blender backface culling must be enabled manually** for the
  outline mesh. Documented in `docs/export/blender.md` — Eevee
  and Cycles default to double-sided.
- **No CI at all yet.** `cargo test` runs on the developer's
  machine. That changes in v0.5.

## Deferred to v0.6

- **C — Palette editor UI** (was v0.4). Right-click swatch to
  edit / delete, "+" slot to add. Demoted because it's polish on
  an already-usable palette and doesn't gate any downstream
  workflow.
- **D — Stroke grouping undo** (was v0.4). `begin_stroke /
  end_stroke` coalescing in `EditHistory`. Demoted for the same
  reason: current per-cell undo is annoying but functional.
- **Greedy rectangle meshing** (was going to be v0.5). The
  culled mesher + per-color buckets is already well inside the
  performance budget; true greedy merging becomes interesting
  only for very large canvases / export size minimisation,
  which are v1.0-adjacent concerns.

## Contract for v0.5 (Headless CLI — new scope)

v0.5 is a full pivot away from GUI polish, triggered by user
request on 2026-04-23. See `NEXT.md` and
`COST_AWARENESS.md` §The Headless Invariant. Summary:

1. Split crate into `lib` (pure headless core) + GUI binary +
   `maquette-cli` binary.
2. `maquette-cli export / info / validate`.
3. `tests/cli.rs` integration suite with committed
   `tests/fixtures/` projects.
4. Stretch: `maquette-cli render` for headless PNG preview
   (visual regression).

After v0.5 lands, the agent can verify export regressions
end-to-end without the user having to open the GUI. User still
validates GUI presentation of new features manually — that's
the division of labour we've agreed to.
