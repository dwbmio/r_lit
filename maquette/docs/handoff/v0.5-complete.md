# v0.5 — headless CLI + lib/bin split

**Status:** shipped. Triggered by user intervention on 2026-04-23
("maquette 支持 headless 么？纯 cli ci 场景很需要"). Closes the
Headless Invariant documented in
`COST_AWARENESS.md` §The Headless Invariant.

## User-visible

- **New binary `maquette-cli`.** `cargo install --path .` now
  installs two executables: `maquette` (GUI, default) and
  `maquette-cli` (headless).
- **`maquette-cli export`** — convert a `.maq` to `.glb` or
  `.gltf` without opening a window. Extension inference, explicit
  `--format`, outline on/off, `--outline-width` (%), and
  `--outline-color #RRGGBB` all supported. Exit code `0` on
  success, `1` on any runtime failure, `2` on usage error (clap
  default).
- **`maquette-cli info`** — print canvas dimensions, painted vs.
  total cell count, max column height, palette size, selected
  color, and the list of color indices actually used. `--json`
  switches to machine-readable output so the command slots
  cleanly into `jq` pipelines and scripted build steps.
- **`maquette-cli validate`** — load a `.maq` file through the
  same code path the GUI uses; exit non-zero on any parse or
  structural error. Suitable as a pre-commit check or CI guard
  for hand-edited project files.

## Under the hood

- **Crate split** (v0.5 sub-task A):
  - `src/lib.rs` — new. Re-exports `grid`, `mesher`, `project`,
    `export`. This is the **headless core**. Compiles without
    `bevy_egui`, `bevy_panorbit_camera`, `bevy_infinite_grid`,
    `bevy_mod_outline`, or `rfd`.
  - `src/main.rs` — GUI binary. Owns only GUI-specific modules:
    `ui`, `camera`, `scene`, `toon`, `history`, `session`,
    `preview_mesh`.
  - `src/bin/maquette_cli.rs` — CLI binary. Depends on the lib
    only; no plugin wiring, no App construction.
- **`project.rs` refactor.** The pure file-format code lives in
  the lib (`maquette::project::{read_project, write_project,
  apply_to_grid_and_palette, ProjectError}`). The GUI-session
  state (`CurrentProject`, `ProjectAction`, the Bevy plugin, the
  rfd dialog calls) moved to `src/session.rs` in the GUI binary.
  CLI and GUI share the I/O code; only the GUI knows about
  "unsaved changes" and window titles.
- **`mesher.rs` refactor.** Pure geometry (`MeshBuilder`,
  `build_color_buckets`, face/normal/quad tables, tests) stays
  in the lib. `MesherPlugin`, `CellMesh`, and the
  `bevy_mod_outline`-integrated rebuild system moved to
  `src/preview_mesh.rs` (GUI-only). The exporter
  (`maquette::export`) now consumes `MeshBuilder` values
  directly without ever constructing a Bevy `Mesh` asset.
- **Exporter stays in the lib.** `export.rs` only uses Bevy ECS
  (`Res`, `Message`, `Plugin`) — no GUI crates. The GUI's
  File → Export menu fires an `ExportRequest` message that the
  plugin handles; the CLI calls `export::write(...)` directly on
  the data it just loaded.
- **Clap 4 CLI parser.** Uses `#[derive(Parser)]` +
  `Subcommand` + `ValueEnum`. Hex-color parser is an internal
  helper (`#RRGGBB` only — we explicitly reject `#rgb`, named
  colors, and CSS-style short forms so the CLI surface stays
  predictable).

## Tests

- **13 unit tests in the lib** (previous 11 + 2 new in
  `project` for the lifted pure I/O code). All pass.
- **6 unit tests in the CLI binary** — hex-color parsing, format
  inference, info summary counting.
- **7 integration tests in `tests/cli.rs`** — shell out to the
  built `maquette-cli` binary via `CARGO_BIN_EXE_maquette-cli`
  (no `assert_cmd` dep). Coverage:
  - GLB export produces a file with the glTF magic + v2 header,
    and the canonical `gltf` crate parses it successfully.
  - glTF (text) export writes a sibling `.bin`.
  - `--no-outline` really removes the outline mesh (fewer
    primitives in the output).
  - `validate` returns success on valid input, failure on
    garbage.
  - `info` returns human text + valid JSON with matching cell
    counts.
  - Missing-input argument returns exit code 1.

These tests are the first CI-ready assertions in the project.
With them, a headless runner can catch export regressions
without human eyes on the GUI.

## Decisions locked this iteration

- **CLI binary name = `maquette-cli`.** GUI stays the default
  `cargo run` target for development ergonomics.
- **Exit codes = `0/1/2`** (success / runtime / usage). Not
  sysexits; chosen for familiarity.
- **Format inference order** — explicit `--format` → file
  extension (`.glb` / `.gltf`) → fall back to GLB.
- **Color syntax = `#RRGGBB` only.** Any other form is an error,
  on purpose, to keep the CLI surface tiny and the parser
  free of edge cases.
- **No `assert_cmd` / `predicates` dev-deps.** `std::process::
  Command` + `std::fs` + `tempfile` covers everything we need
  right now and keeps `cargo test` fast for an offline dev box.
- **No checked-in `.maq` fixtures** — tests build their
  fixtures in-memory via `Grid::with_size` + `paint`, then
  write them to a `tempfile::tempdir()`. Keeps the repo clean
  and sidesteps fixture-drift when the schema evolves. We can
  add real on-disk fixtures in v0.6 if visual regression
  work needs them.

## Known issues / caveats

- **Release build of the CLI still compiles `bevy`, `bevy_winit`,
  `bevy_egui`, etc.** transitively via the `[dependencies]`
  table. Cargo's unit-of-compilation is the crate, not the
  binary; splitting features to make the CLI a lean binary is
  tracked informally but isn't worth the Cargo-feature-flag
  cognitive overhead until the build time hurts.
- **No `render` verb yet.** `maquette-cli render` for headless
  PNG preview (visual regression) was a v0.5 stretch; deferred
  to v0.6 or later. The export-path tests already catch
  structural regressions; render-diff tests would catch shader
  regressions, which we don't have yet.
- **The GUI's `handle_shortcuts` / `handle_export_request`** now
  live in different modules than the code they drive. The new
  module boundary is intentional (Headless Invariant), but it
  does mean a grep for "export" touches more files. Tradeoff.

## Contract for v0.6

v0.6 picks up the GUI polish items demoted from v0.4 plus the
CLI stretch goal:

1. **C from old v0.4** — palette editor UI (right-click swatch,
   "+" slot). The CLI will not grow a palette-edit verb; palette
   authoring is interactive.
2. **D from old v0.4** — stroke grouping in `EditHistory`
   (`begin_stroke / end_stroke`). Pure data change; deserves a
   CLI-callable test even though there's no `maquette-cli undo`
   verb. (We test the data structure directly.)
3. **G from v0.5 (stretch, now required)** — `maquette-cli
   render` for headless PNG preview. Requires either Bevy's
   headless plugin group + offscreen target, or a minimal custom
   wgpu setup. If it stays flaky/expensive, we ship a narrower
   version that renders only the T-view from the UI spec.
4. **Greedy rectangle meshing** in `mesher.rs`. Collapses
   same-color coplanar quads. Drops export file size roughly
   by 5–10× for large canvases. Reuses the existing `MeshBuilder`
   output shape so nothing downstream changes.

v1.0 is then primarily about doc + release polish: engine
integration guides expand with video walkthroughs, CI gets a
matrix (macOS / Windows / Linux), and we write the launch story.
