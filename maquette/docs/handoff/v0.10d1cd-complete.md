# v0.10 D-1.C / D-1.D — Generate-all + Textured render — Complete

Status: **shipped on `main`** (this commit).

This handoff finishes the v0.10 D-1 milestone:

* **D-1.C** — Material drawer's `Generate all (N live slots):` row.
  Three buttons (Mock / CPU / Fal) walk the live palette and queue
  one `GenerateSlotTexture` event per slot for the chosen lane.
  Existing per-slot busy guard from D-1.B drops duplicates so a
  second click while N tasks are in flight is a no-op.
* **D-1.D** — `View: Flat / Textured` toggle now actually flips
  the rendered output. `ToonMaterial` grew an optional
  `base_color_texture: Option<Handle<Image>>` ; `assets/shaders/
  toon.wgsl` samples it and multiplies against `base_color × shade`;
  a new `texture_registry` module loads `<cache_key>.png` into
  Bevy `Image` assets on demand and reuses the same handle across
  every palette slot that points at it.

D-1 as a whole (A + B + C + D) is now closed.

## What landed

### D-1.C — `Generate all` × 3 lanes

`material_drawer()` (`src/ui.rs`) gained a row at the bottom:

```
Generate all (12 live slots):
[ Mock ] [ CPU ] [ Fal ]
```

Each button is `disabled` when `palette.iter_live().count() == 0`
(empty palette → friendly "add a colour first" disabled-hover).
Clicking enumerates every live slot and writes one
`GenerateSlotTexture { slot, provider }` event per slot. The
per-slot busy guard in `slot_texgen::SlotTexgenState::in_flight`
already drops duplicate requests, so a second click while N tasks
are in flight is a no-op rather than a stampede.

Why **not** Rustyme Canvas group + chord callback (sonargrid does
support it, see `protocol.rs::TaskEnvelope::group_id` /
`chord_callback`):

* Chord callback batches all PNG results into a single reply
  envelope. We'd lose the "this slot is done now → flash a badge,
  cube swaps to texture" progressive feedback win that comes for
  free with N independent tasks.
* If "atomic apply all or nothing" semantics ever become a
  product requirement, switching to chord callback is a contained
  change (`SlotTexgenPlugin` would reuse most of its existing
  state machine).

### D-1.D — Textured render through the toon shader

#### `ToonMaterial` (`src/toon.rs`)

```rust
#[derive(Asset, TypePath, AsBindGroup, Debug, Clone, Default)]
pub struct ToonMaterial {
    #[uniform(0)]
    pub params: ToonParams,
    #[texture(1)]
    #[sampler(2)]
    pub base_color_texture: Option<Handle<Image>>,
}
```

`Option<Handle<Image>>` exploits Bevy's
`AsBindGroup`-Option semantics: when `None`, Bevy inserts a 1×1
white image as a fallback so the shader can sample
unconditionally. Result: **a single shader path covers both Flat
and Textured modes** — no `#ifdef`, no per-mode pipeline.

Two constructors, both used by `preview_mesh`:

* `ToonMaterial::with_color(color)` — Flat (no texture handle).
* `ToonMaterial::with_color_and_texture(tint, handle)` — Textured
  (handle present + tint=White, so the texture dominates).

#### Shader (`assets/shaders/toon.wgsl`)

```wgsl
@group(#{MATERIAL_BIND_GROUP}) @binding(0) var<uniform> material: ToonParams;
@group(#{MATERIAL_BIND_GROUP}) @binding(1) var base_color_texture: texture_2d<f32>;
@group(#{MATERIAL_BIND_GROUP}) @binding(2) var base_color_sampler: sampler;

@fragment
fn fragment(in: VertexOutput) -> @location(0) vec4<f32> {
    // ... toon stepping unchanged ...
    let tex = textureSample(base_color_texture, base_color_sampler, in.uv);
    let albedo = material.base_color.rgb * tex.rgb;
    let alpha  = material.base_color.a   * tex.a;
    return vec4<f32>(albedo * shade, alpha);
}
```

Pixel-output equivalence in Flat mode is preserved by the white
fallback texture: `base_color × 1.0 × shade == pre-D-1.D output`.

#### `texture_registry.rs` — cache_key → Handle<Image>

New 230-line bin-only module wrapping a `Resource:
TextureRegistry { entries: HashMap<String, LoadState> }`.

```
preview_mesh::rebuild_cell_mesh
   │
   ▼  for each palette bucket (slot_idx, color):
TextureRegistry::handle_for(cache_key, &mut Assets<Image>)
   │
   ├─ HashMap hit (Loaded) → clone existing Handle<Image>
   ├─ HashMap hit (Failed) → None (sticky failure, no retry)
   └─ HashMap miss        → resolve_and_load:
                              read ~/.cache/maquette/textures/<key>.png
                              → decode_png_to_image (RGBA8 sRGB)
                              → Assets<Image>::add → Handle
                              → cache as Loaded(handle)
```

Sticky failures matter: a project referencing a missing or corrupt
PNG would otherwise re-decode every frame the user is editing.
Caching the failure means one warn-log per missing cache_key per
session and zero per-frame overhead.

`decode_png_to_image` is the function that previously lived in
`block_composer.rs` — now centralised here so the slot-texgen
pipeline and the Composer share the same decoder.

#### `preview_mesh.rs` — material selection

`rebuild_cell_mesh` grew two changes:

1. **Trigger expansion** —

   ```rust
   let needs_rebuild = grid.dirty
       || palette.is_changed()
       || meta.is_changed();
   ```

   The original `grid.dirty` only fires on paint strokes / size
   changes. A `slot_texgen` task finishing populates
   `PaletteSlotMeta::texture` and bumps `palette` change-tick;
   flipping `View: Flat / Textured` mutates `meta` and bumps its
   change-tick. Either now triggers a rebuild.

2. **`build_slot_material` helper** — given `(palette, slot_idx,
   color, textured, registry, images)`, picks one of three
   materials in this priority order:

   * Flat view → `with_color(color)` (no texture handle).
   * Textured view + slot has `cache_key` that loads OK →
     `with_color_and_texture(WHITE, handle)`.
   * Textured view + missing meta / no cache_key / decode failed
     → fallback to `with_color(color)` — **never magenta missing-
     texture aesthetic**, the user sees the slot's flat colour
     until they generate that slot.

The fallback is what makes Textured mode work *progressively* :
slots without textures show their flat colour (looks fine),
slots with textures show them. The user can generate slots one
at a time and watch the preview fill in.
