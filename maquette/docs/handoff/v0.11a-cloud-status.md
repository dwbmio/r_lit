# v0.11 A — Cloud Status Chip — Complete

Status: **shipped on `main`** (this commit).

This is the first Cloud-as-Backup slice. It deliberately does **not**
change any project IO path: File → Open / Save / Export stay local
first. The only shipped behavior is a short hfrog availability probe
and a status-bar chip that makes the current mode visible.

## What Landed

### `cloud_status.rs`

New GUI-only module with:

* `CloudStatusPlugin`
* `CloudStatus` resource
* `AppCloudMode::{Probing, Online, Offline}`
* `ProbeCloud` message
* one startup probe
* click-to-reprobe support

Probe target:

```text
<MAQUETTE_HFROG_BASE_URL>/api/artifactory/list?runtime=ping
```

Timeout:

```text
1500 ms
```

That timeout is intentionally short. Cloud availability is a UX hint,
not a blocker; if hfrog is slow, the app shows `Local mode` quickly and
the user keeps working.

### Status Bar UI

`ui.rs` now renders a small chip at the right side of the bottom status
bar, next to the version label:

| Mode | Chip |
|------|------|
| probing | `Cloud: checking...` |
| online | `Cloud: online` |
| offline | `Local mode` |

The chip has a hover tooltip with the hfrog base URL, last checked
timestamp, and either the success state or the last failure. Clicking it
writes `ProbeCloud` and reruns the short probe. While a probe is already
in flight, the chip is disabled and says `Cloud probe already running`.

### Main Wiring

`main.rs` registers:

```rust
.add_plugins(cloud_status::CloudStatusPlugin)
```

`ui.rs` keeps `Res<CloudStatus>` inside the existing `ProjectState`
`SystemParam` bundle so `ui_system` does not exceed Bevy ECS's
16-parameter ceiling again.

## Explicit Non-Goals

This slice does **not**:

* upload `.maq` files
* download cloud projects
* change Save / Open / autosave behavior
* merge local and cloud Recent lists
* auto-sync block metadata
* upload texture cache PNGs

All of that remains in v0.11.B/C/D. This slice only establishes the
mode vocabulary and the "cloud reachable vs local mode" UI.

## Manual Check

`USER-TODO.md` now has `#cloud-chip`:

1. Normal network: start GUI → `Cloud: checking...` → `Cloud: online`
   within about 1-2 seconds.
2. Hover chip → tooltip shows `https://hfrog.gamesci-lite.com` and
   last checked timestamp.
3. Click chip → reprobe.
4. Bad URL:

   ```sh
   MAQUETTE_HFROG_BASE_URL=http://127.0.0.1:1 cargo run --bin maquette
   ```

   should settle into `Local mode` quickly; Save / Open still work
   locally.
